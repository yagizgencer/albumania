## Deploy steps (verbatim — run once per environment)

### 1. Neon (Postgres)

1. Sign up at https://neon.tech
2. New Project → choose a region → copy the **connection string** (starts with `postgresql://`)
3. Keep it for the Render env vars step.

### 2. Cloudflare R2 (avatar storage)

1. Sign up at https://cloudflare.com
2. Left nav → **R2** → **Create bucket** → name it `albumania-avatars`
3. On the bucket page → **Settings** → **Public Access** → enable it. Copy the **Public Bucket URL** (looks like `https://pub-xxxx.r2.dev`)
4. Left nav → **R2** → **Manage R2 API Tokens** → **Create API Token**
   - Permissions: **Object Read & Write** on bucket `albumania-avatars`
   - Copy **Account ID**, **Access Key ID**, **Secret Access Key**

### 3. Render (FastAPI backend)

1. Sign up at https://render.com
2. **New → Web Service → Connect a repository** → select `yagizgencer/albumania`
3. Root directory: `backend` — Render will pick up `render.yaml` automatically
4. Under **Environment Variables** add:

| Key | Value |
|---|---|
| `DATABASE_URL` | The Neon connection string (with `?sslmode=require` at the end) |
| `SPOTIFY_CLIENT_ID` | Your Spotify app client ID |
| `SPOTIFY_CLIENT_SECRET` | Your Spotify app client secret |
| `CORS_ORIGINS` | Your Vercel URL (add after Vercel deploy; put `http://localhost:5173` first) |
| `API_BASE_URL` | The Render service URL (e.g. `https://albumania-api.onrender.com`) |
| `R2_ACCOUNT_ID` | From step 2 |
| `R2_ACCESS_KEY_ID` | From step 2 |
| `R2_SECRET_ACCESS_KEY` | From step 2 |
| `R2_BUCKET` | `albumania-avatars` |
| `R2_PUBLIC_URL_BASE` | The public bucket URL from step 2 |

> `JWT_SECRET` is auto-generated by Render (declared with `generateValue: true`).
> `STORAGE_BACKEND` is hardcoded to `r2` in `render.yaml`.

5. Click **Deploy**. First deploy runs `alembic upgrade head` then starts uvicorn.
6. After deploy completes, visit `https://<your-render-url>/health` → should return `{"status":"ok"}`.

### 4. Vercel (React frontend)

1. Sign up at https://vercel.com
2. **Add New → Project → Import Git Repository** → select `yagizgencer/albumania`
3. **Root Directory**: `frontend`
4. Framework preset: **Vite** (auto-detected)
5. Add environment variable:
   - `VITE_API_BASE_URL` = `https://<your-render-url>` (no trailing slash)
6. Click **Deploy**. Vercel reads `frontend/vercel.json` and sets the SPA rewrite.
7. Copy the Vercel URL (e.g. `https://albumania.vercel.app`).

### 5. Wire CORS

Back in Render → Environment → update `CORS_ORIGINS` to:
```
https://albumania.vercel.app,http://localhost:5173
```
Render redeploys automatically.

### 6. Spotify developer app

If you haven't already:
1. Go to https://developer.spotify.com/dashboard → **Create app**
2. Redirect URIs: add anything (we use Client Credentials, no redirect needed)
3. Copy **Client ID** and **Client Secret** → paste into Render env vars

### 7. End-to-end smoke test

- Register two accounts on the live URL
- Friend each other
- Both rate the same album from search
- Confirm it appears in the friend dashboard with the correct date
- Toggle one profile to `private` — confirm a stranger gets a 403

## Notes

- **Cold start**: Render free tier idles after 15 min. First request after idle takes ~15–30 s. Acceptable for MVP.
- **Alembic on deploy**: `preDeployCommand` in `render.yaml` runs migrations before the new instance goes live — safe zero-downtime migrations.
- **R2 vs local**: `STORAGE_BACKEND=r2` is hardcoded for production. Local dev still uses `local` (default in `config.py`).
- **JWT_SECRET**: `generateValue: true` in `render.yaml` means Render creates a secure random value on first deploy and persists it. Don't override it unless you want to invalidate all existing sessions.
